6. 처리프로세스​

6.1 입력항목검증​

  • 상품코드는 요구불상품만 가능
    • 상품유형, 상품템플릿 고정
  • 비밀번호 정당성 체크
    • 비밀번호, 비밀번호2의 일치여부 확인
  • 과세구분이 세금우대, 생계형인 경우는 세금우대한도금액을 입력해야 함
    • 저축은행은 생계형만 사용
  • 연동출금 시 계좌번호, 비밀번호, 출금금액 모두 입력해야 함
    • 출금금액은 대체금액을 초과할 수 없음

6.2 계약원장생성​

  • 관련 API : openArr

6.3 서비스기본검증​

  • 관련 API : validate, doServiceAction

6.4 거래명세작성​

  • 거래금액 Entry 정보를 조립하여 계정처리를 함
    • 거래명세를 하나로 만들기 위함
      • 기존은 입금서비스 연동처리함 : 신규명세(0원), 입금명세(신규금액) 2개 생성
  • 관련 API : createArrTx

6.5 통장발행​

  • 통장발급을 하는 경우
  • 통장발행서비스를 호출하여 출력항목정보를 받음
  • 관련 서비스 : 통장발행(SCBSDPTM01100)

6.6 연동출금처리​

  • 연동출금을 하는 경우
  • 출금서비스를 호출
  • 관련서비스 : 요구불지급(SCBSDPFT00900)
